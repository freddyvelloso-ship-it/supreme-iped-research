## Acceptance Surface

- CI/CD runs readiness, release, security scans, SBOM, Phase 2-7 gates, tests,
  Docker builds and image digest artifacts.
- Staging uses the same production compose stack plus `docker-compose.staging.yml`
  for isolated volumes, staging ports and staging env templates.
- Backup/restore is tested by restoring each Postgres dump into a temporary
  database, not by overwriting the working database.
- Observability includes availability and latency SLOs, burn-rate alerts, DLQ
  objectives, backup/restore objectives and runbooks.
- Developer, researcher, auditor and operator documentation exists.
- External security and statistical audit packages exist.
- A NIST CFTT-inspired benchmark is reproducible and records metrics.
- A technical whitepaper exists with limits and responsible-use boundaries.
- Final release ZIP is clean and is tested after extraction in staging.
