# SUPREME IPED Plugin

This module is the Phase 8 replacement path for the old source patch.

It implements `iped.viewers.api.ResultSetViewer`, which IPED loads from
`conf/ResultSetViewersConf.xml` while plugin JARs are available through the
`plugins` classpath. The plugin observes the IPED results table and writes
privacy-minimized NDJSON events for the local SUPREME agent.

Production status:

- Plugin architecture: implemented.
- Build artifact: generated by `scripts/build_supreme_iped_plugin.ps1`.
- Real code-signing certificate: required for production and not bundled.
- Current development signing mode: hash manifest only unless a signing
  keystore is supplied through environment variables.

The legacy `supreme-audit-patch.jar` remains historical compatibility only and
must not be treated as the final field architecture.
